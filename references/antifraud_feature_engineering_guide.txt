# 反欺诈交易特征工程干货大全
# 来源：学术论文(Nature/arXiv)、Kaggle竞赛Top方案、Medium、行业实践
# Todd每轮参考，灵活选用。BCA项目只有流水表(txn_tmp2)和标签表(txn_label_tmp1)。
# 最大回溯150天(12960000秒)。

# ============================================================
# 一、RFM框架（学术界+业界公认最基础的特征体系）
# ============================================================
# 来源：Baesens et al. 2015, Van Vlasselaer et al. 2015
# Nature论文(Xie 2022)实验证明：monetary类特征F1>50%，frequency/recency类单独用很差
# 但组合起来效果最好。
#
# R (Recency) 近因性：
# - 距上次出金的秒数/小时数
# - 距上次入金的秒数
# - 距上次使用同一渠道的时间
# - 距上次向同一对手转账的时间
#
# F (Frequency) 频率：
# - 过去1h/1d/7d/14d/150d的出金笔数
# - 过去1h/1d/7d/14d/150d的入金笔数
# - 过去1d/7d/14d的不同对手数(nunique dfzh)
# - 过去1d/7d/14d的不同渠道数(nunique tranchan)
#
# M (Monetary) 金额：
# - 过去14d/150d出金总额、均值、中位数、最大值、标准差
# - 过去14d/150d入金总额、均值
# - 当笔金额 / 14d出金均值 (z-score思想)
# - 当笔金额 / 14d出金中位数 (更稳健)
# - 当笔金额 / 14d出金最大值
# - 当笔金额 / 150d出金最大值

# ============================================================
# 二、Velocity特征（速度/加速度）
# ============================================================
# 来源：DigitalOcean 2025, Kaggle Top方案, KPMG 2025
#
# - 过去1小时内交易笔数（短时爆发）
# - 过去24小时内交易笔数
# - 3天出金笔数 / 14天出金笔数 = 短期加速比
# - 3天出金金额 / 14天出金金额 = 金额加速比
# - 7天出金笔数 / 150天出金笔数 × (150/7) = 频率相对加速度
# - 当天交易笔数 / 过去14天日均交易笔数 = 当日爆发度
# - 连续N笔交易的时间间隔均值和标准差

# ============================================================
# 三、偏离度特征（Z-score / 异常度）
# ============================================================
# 来源：NumberAnalytics 2025, HighRadius 2024, arXiv 2025
#
# - (当笔金额 - 14d出金均值) / 14d出金标准差 = 金额z-score
# - (当笔金额 - 150d出金均值) / 150d出金标准差
# - (当天出金笔数 - 14d日均出金笔数) / 14d日标准差 = 频率z-score
# - 当笔金额在14d出金金额中的百分位排名 (percentile_rank)
# - 当笔金额是否超过14d出金金额的95分位 (0/1标志)

# ============================================================
# 四、时间模式特征
# ============================================================
# 来源：Bahnsen et al. 2016 (von Mises分布), Kaggle IEEE-CIS
# Bahnsen的论文证明加入周期性时间特征后savings提升13%
#
# - hour_sin = sin(2π * hh/24), hour_cos = cos(2π * hh/24) → 周期编码
# - is_night = (hh >= 22 or hh <= 5) → 夜间标志
# - is_weekend (如果有星期几信息)
# - 当笔交易小时 vs 该账户历史最常交易小时的偏差
# - 过去14天夜间出金占比
# - 过去14天凌晨(0-5点)出金占比
# - 连续两笔交易的时间间隔(秒)

# ============================================================
# 五、渠道/终端分布特征（暴力枚举）
# ============================================================
# 来源：SD项目规则SQL经验 + 特征工程策略指南
#
# - 遍历tranchan所有取值，每种渠道过去14d/150d的出金次数
# - 每种渠道的出金金额占总出金的比例
# - 渠道多样性：过去14d用了多少种不同渠道 (nunique)
# - 渠道集中度：最常用渠道的出金占比 (top1_prop)
# - 渠道熵值：-Σ(p_i * log(p_i))，越高越分散
# - 当笔渠道在历史中的使用频率排名
# - 当笔渠道是否在14d/150d内首次出现 (is_new_channel)
# - termtype同理遍历

# ============================================================
# 六、对手(counterparty)网络特征
# ============================================================
# 来源：APATE论文(Van Vlasselaer 2015), MuleTrace 2025, Nature 2022
#
# - 过去14d/150d不同出金对手数
# - 过去14d/150d不同入金来源数
# - 入金来源集中度：最大入金来源占总入金的比例
# - 出金对手集中度：最大出金对手占总出金的比例
# - 当笔对手是否在14d/150d内首次出现
# - 当笔对手在历史中出现过多少次
# - 入金来源数 / 出金对手数 的比值（少入多出=可疑）
# - 共享对手：该对手在同期被多少不同账户转入过（需要全局视角，BCA可能做不了）

# ============================================================
# 七、金额分档特征（暴力枚举）
# ============================================================
# - 分6档：<=100, 100-1k, 1k-3k, 3k-1w, 1w-5w, >5w
# - 每档过去14d/150d的出金笔数
# - 每档的出金金额占比
# - 当笔金额所在档位是否在14d/150d内首次出现 (famt_bin陌生标志)
# - 小额(<100)出金占比
# - 大额(>1w)出金占比
# - 小额出金笔数的短期加速度

# ============================================================
# 八、入金→出金关联特征（受害者账户核心模式）
# ============================================================
# 来源：SD项目规则SQL (lag_in系列, rn_out_follow_in)
# 这是受害者账户最典型的模式：先收到被骗款(入金)，然后立即转走(出金)
#
# - 最近一笔入金的金额
# - 当笔出金到最近入金的时间间隔(秒)
# - 间隔越短越可疑（钱刚进来就转走）
# - 最近入金后的出金笔数 (rn_out_follow_in)
# - 最近入金金额 / 当笔出金金额 的比值
# - 最近入金是否来自陌生对手 (lag_in_fnwp2)
# - 最近入金后24小时内的出金总额 / 入金金额

# ============================================================
# 九、余额相关特征
# ============================================================
# 来源：SD项目规则SQL (txn_adj_c_zhye_min_7200s)
# 注意：需要确认BCA的txn_tmp2是否有余额(zhye)字段
#
# - 入金后2小时内最低余额（迅速被转走的信号）
# - 入金后7天内最低余额
# - 当笔交易后余额 / 入金前余额
# - 如果没有余额字段，可以用累计入金-累计出金近似

# ============================================================
# 十、Kaggle IEEE-CIS竞赛Top方案技巧
# ============================================================
# 来源：Chris Deotte (1st place), kozodoi (top 3%)
#
# 1. UID构建：从多个字段组合出唯一客户ID，然后基于UID做聚合特征
#    BCA已有cardno作为客户ID
#
# 2. 频率编码(frequency encoding)：
#    将分类变量替换为其出现频率
#    例：tranchan='MB TRANSFER' → 该渠道在全量数据中的出现次数/占比
#
# 3. Target编码(target encoding)：
#    将分类变量替换为该类别下label=1的比例
#    例：tranchan='VA DB' → 该渠道下欺诈交易的占比
#    注意：必须用训练集统计，不能泄漏测试集信息
#
# 4. 聚合特征(group aggregation)：
#    groupby cardno → count/mean/std/max/min/nunique
#    groupby tranchan → count/mean/std
#    groupby (cardno, tranchan) → count/sum
#
# 5. 交叉特征：
#    金额z-score × 是否陌生对手
#    夜间标志 × 小额标志
#    新渠道 × 新对手 × 夜间
#    渠道集中度 × 对手集中度

# ============================================================
# 十一、高级技巧
# ============================================================
#
# 1. 滑动窗口多粒度：不只用14d和150d，加1d/3d/7d/30d/90d
#    不同窗口捕捉不同时间尺度的变化
#
# 2. 差分特征：7d指标 - 14d指标，捕捉趋势变化
#    例：7d日均出金次数 - 14d日均出金次数 = 近期加速
#
# 3. 比率特征：短窗口 / 长窗口
#    例：3d出金次数 / 14d出金次数，越接近1说明活动越集中在最近
#
# 4. 熵特征：对分类字段算信息熵
#    渠道熵、对手熵、金额档熵
#    熵低=行为集中=可疑（正常人渠道多样，欺诈往往固定一两个渠道）
#
# 5. 连续性特征：
#    连续N笔出金间隔的标准差（间隔稳定=自动化转账=可疑）
#    连续N笔出金金额的标准差（金额稳定=批量操作）
#
# 6. 首末特征：
#    当天第一笔交易 vs 最后一笔交易的时间差
#    当天第一笔金额 vs 最后一笔金额
